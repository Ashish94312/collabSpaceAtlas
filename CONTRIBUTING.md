# Contributing to CollabSpaceAtlas

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone <your-fork-url>`
3. Create a virtual environment and install dependencies
4. Create a feature branch: `git checkout -b feature/your-feature-name`

## Content Guidelines

### Adding New Content

1. Choose the appropriate topic folder in `content/topics/`
2. Create a new markdown file with a descriptive name
3. Use the templates in `content/templates/` as a starting point
4. Follow the frontmatter format exactly
5. Ensure content is well-structured and properly formatted

### Content Quality

- Write clear, concise, and accurate content
- Include practical examples where applicable
- Add proper headings and structure
- Include references and further reading
- Test that markdown renders correctly

## Code Contributions

### Style Guidelines

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions focused and modular
- Write clear comments for complex logic

### Testing

- Test your changes locally before submitting
- Ensure all migrations run successfully
- Verify content loads and displays correctly
- Check that the sidebar navigation works

## Pull Request Process

1. Update the README if needed
2. Ensure your code follows style guidelines
3. Test thoroughly
4. Submit a pull request with a clear description
5. Respond to any feedback or requested changes

## Questions?

Feel free to open an issue for questions or discussions about the project.
